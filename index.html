What Is Method Overloading?

When two or more methods within the same class are declared with the same name, but different parameters they are said to be Overloaded.
When an overloaded method is invoked, java uses its return type or number of arguments to determine which version to call. While overloaded methods may have different return types, the return type alone is insufficient to distinguish two version of a method. When java encounters a call to an overloaded method, it simply executes the version of the method whose parameters match the arguments used in the call. However, this match need not always be exact. In some cases Java's automatic type conversions can play a role in overloading resolution. Let's take an example to better understand the concept.

Example:

         class A
         {
             void specify(){
             System.out.println("No parameter");
             }
                     
             //Overload specify for two integer parameters.
             void specify(int a, int b){
             System.out.println("a and b: " + a + " " + b);
             }
                         
             //Overload specify for a double parameter.
             void specify(double a){
             System.out.println("Inside spsecify(double) a: " + a);
             }
         }
                         
         class B{
             public static void main(String args[])
                 {
                     A ab = new A();
                     int i = 11;
                         
                     ab.specify();
                     ab.specify(10,20);
                         
                     ab.specify(i); //this will invoke test(double)
                     ab.specify(119.72);
                 }
             }

Console Output:

         No parameter
         a and b: 10 20
         Inside spsecify(double) a: 11.0
         Inside spsecify(double) a: 119.72

In the program, class A does not define specify(int). Therefore when specify() is called with an integer argument inside A, no matching method is found. However, java automatically converts integer into a double and this coversion resolves the call. Therefore, after specify(int) is not found, java elevates i to double and then calls specify(double). Of course, if specify(int) had been defined, it would have been called instead. Thus, automatic conversion is only applied if no match is found.
Hence, method overloading supports polymorphism bacause it is one way java implements the one interface, multiple methods paradigm.

What Is Method Overriding?

In class heirarchy, when a method in a subclass has the same name and type signatures as a method in its superclass, it is said to be Overriden. When an overidden method is called from within a subclass, it will always refer to the version of method defined by the subclass. The version of method defined by superclass will be hidden. Let's understand this with an example:

Example:

         class A{
             int i;
             int j;
         
                 A(int a, int b)
                 {
                     i = a;
                     j = b;
                 } 
                     
             //Display i and j
                     
             void show(){
                 System.out.println("i and j: " + i + " " + j);
             }
                     
         class B extends A {
             int k;
                     
             B(int a, int b, int c){
                 super(a,b);
                 k = c;
             }
                     
             //Display k - this overrides show() in A 
             @Override
             void show(){
             System.out.println("k: " + k);
             }
         }
                     
         class Overriding{
             public static void main(String[]args){
                 B ob = new B(2,3,4);
                     
                 ob.show(); // this calls show in B
             }
         }

Console Output:

k: 4

In the above program when show() is invoked on an object of type B, the version of show() within B is used. That is, the version of show() inside B overrides the version declared in A.
If you wish to access the superclass version of an overriden function, you can do it by using keyword super. For example in the above program, the superclass version of show() is invoked within the subclass' version. this allow all instance variable to be displayed.

Modification in class B:

         class B extends A{
             int k;
                        
             B(int a, int b, int c)
             {
                 super(a,b); //super used to override constructor
                 k = c;
             }
         
             @Override
             void show(){
                 super.show(); // this call's a show()
                 System.out.println("k: + k");
             }
         }

Console Output:

         i and j: 1 2
         k: 3

In a complex programs, where different types of objects are refferd to, different version of an overriden method will be called. To resolve such issue we'll use dynamic method dispatch. Dynamic method dispatch is the mechanism by which a call to an overrideen method is resolved at run time, this is how java implements run-time polymorphism. It uses the fact:a superclass reference variable can refer to a subclass object. Let's take a sample program.

Example:

         class A{
                         
             void show(){
                 System.out.println("inside A's show method");
                 }
         }
                         
         class B extends A {
                           
             @Override
             void show(){
                 System.out.println("Inside B's show method");
                 }
         }
                         
         class C extends A {
                           
             @Override
             void show(){
                 System.out.println("Inside C's show method");
                 }
         }
                         
         class dynDispatch{
             public static void main(String[]args){
                 A a = new A(); //Object of type A
                 B b = new B(); //Object of type A
                 C c = new C(); //Object of type A
                 A r; //Obtain a reference of type A
                         
                 r = a; // r refer to an A object
                 r.show(); //calls A's version of show
                         
                 r = b; // r refer to an B object
                 r.show(); //calls B's version of show
                         
                r = c; // r refer to an C object
                r.show(); //calls C's version of show
                         
             }
         }

Console Output:

         inside A's show method
         Inside B's show method
         Inside C's show method
